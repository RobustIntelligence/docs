# ri/api/project/project.proto
## Version: version not set

### /v1/projects

#### GET
##### Summary:

ListProjects

##### Description:

Returns a paginated list of Projects for a given Workspace ID. 
Filters out Projects that the user does not have access to.
The list is sorted by the last test run time field of each Project.

##### Parameters

| Name | Located in | Description | Required | Schema |
| ---- | ---------- | ----------- | -------- | ---- |
| workspaceId.uuid | query | String value of the object ID. | No | string |
| pageToken | query | Specifies a page of the list returned by a ListProjects query. The ListProjects query returns a pageToken when there is more than one page of results. Specify either this field or the firstPageQuery field. | No | string |
| pageSize | query | The maximum number of Project objects to return in a single page. | No | string (int64) |

##### Responses

| Code | Description | Schema |
| ---- | ----------- | ------ |
| 200 | A successful response. | [projectListProjectsResponse](#projectlistprojectsresponse) |
| default | An unexpected error response. | [googlerpcStatus](#googlerpcstatus) |

#### rimeActorRole

ActorRole specifies a set of permissions an actor has on an object.

 - ACTOR_ROLE_ADMIN: Admin role has full manage access to the object, including deletion
and adding users.
 - ACTOR_ROLE_USER: User role has edit and view access to the object.
 - ACTOR_ROLE_SUPPORT: Support role is used by RI support engineers.
 - ACTOR_ROLE_VIEWER: Viewer role is read-only.
 - ACTOR_ROLE_NONE: None role has no permissions to the object.
 - ACTOR_ROLE_VP: VP role has full manage access to the object, including deletion
and adding users.

| Name | Type | Description | Required |
| ---- | ---- | ----------- | -------- |
| rimeActorRole | string | ActorRole specifies a set of permissions an actor has on an object.   - ACTOR_ROLE_ADMIN: Admin role has full manage access to the object, including deletion and adding users.  - ACTOR_ROLE_USER: User role has edit and view access to the object.  - ACTOR_ROLE_SUPPORT: Support role is used by RI support engineers.  - ACTOR_ROLE_VIEWER: Viewer role is read-only.  - ACTOR_ROLE_NONE: None role has no permissions to the object.  - ACTOR_ROLE_VP: VP role has full manage access to the object, including deletion and adding users. |  |
